# نظام إدارة مبيعات ورشة أبو أحمد للدراجات الهوائية

سيرفر Node.js + Express متصل بقاعدة بيانات PostgreSQL حقيقية، مصمم للنشر على Railway.
كل البيانات (المنتجات والمبيعات) محفوظة في قاعدة البيانات فعلياً — لا بيانات وهمية، ولا Local Storage.

## محتويات المشروع
- `server.js` — السيرفر وواجهة API (المنتجات، المبيعات، التقارير)
- `public/index.html` — واجهة النظام (نفس تصميمك الأصلي)
- `package.json` — الاعتماديات
- `.env.example` — نموذج لمتغيرات البيئة المطلوبة

## خطوات النشر على Railway

### 1. رفع المشروع
ارفع محتويات هذا المجلد إلى مستودع على GitHub (أو استخدم Railway CLI بدون GitHub: `railway login` ثم `railway init` ثم `railway up` من داخل هذا المجلد).

### 2. إنشاء المشروع على Railway
1. من لوحة Railway: **New Project → Deploy from GitHub repo** واختر المستودع.
2. انتظر أول عملية Build (ستفشل مؤقتاً لعدم وجود قاعدة بيانات بعد — طبيعي).

### 3. إضافة قاعدة بيانات PostgreSQL
1. بنفس المشروع: **+ New → Database → Add PostgreSQL**.
2. Railway ينشئ قاعدة البيانات تلقائياً بمتغير اسمه `DATABASE_URL`.

### 4. ربط قاعدة البيانات بالسيرفر
1. افتح خدمة السيرفر (وليس قاعدة البيانات) → تبويب **Variables**.
2. أضف متغير جديد:
   - الاسم: `DATABASE_URL`
   - القيمة: اختر **Add Reference** واربطه بـ `Postgres.DATABASE_URL` (يظهر تلقائياً بالقائمة).

### 5. حماية النظام (مهم لأنه خاص بمتجرك)
بنفس تبويب Variables أضف:
- `APP_USER` = اسم مستخدم تختاره
- `APP_PASSWORD` = كلمة مرور قوية تختارها

بدون هذين المتغيرين، أي شخص يعرف الرابط يمكنه فتح النظام والتعديل عليه.

### 6. النشر والوصول
1. بعد حفظ المتغيرات، Railway يعيد النشر تلقائياً.
2. من تبويب **Settings → Networking → Generate Domain** تحصل على رابط عام (مثل `xxx.up.railway.app`).
3. افتح هذا الرابط من أي جهاز (موبايل، لابتوب، أكثر من جهاز بنفس الوقت) — سيطلب اسم المستخدم وكلمة المرور، وكل البيانات مشتركة وحقيقية من قاعدة البيانات نفسها.

## ملاحظات
- كل عملية بيع تُحفظ فوراً في قاعدة البيانات لحظة الضغط — لا يوجد خطر ضياع بيانات عند إغلاق الجهاز.
- كل يوم يُخزَّن بتاريخه الخاص، فالأرشفة اليومية (منتصف الليل) تلقائية بدون أي إجراء إضافي.
- التقارير تدعم: اليوم، هذا الأسبوع، هذا الشهر، أو تاريخ مخصص.
- لعمل نسخة احتياطية من قاعدة البيانات: من لوحة Railway بخدمة Postgres يوجد خيار **Backups**.
